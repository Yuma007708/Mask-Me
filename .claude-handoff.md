## 未コミット変更（plan 実装済み、検証後にコミット予定）

```
M App/MaskMe/MaskMeApp.swift                                  ← DValid 一時投入（コミット前に削除）
M App/MaskMe/Model/MediaPipeFaceLandmarkerAdapter.swift       ← eyeRatio 0.40 + bbox サイズ/比ガード
M App/MaskMe/Model/MosaicPreviewController.swift              ← itemTimeForDisplay 同期
M App/MaskMe/Model/YuNetFaceDetector.swift                    ← letterbox リサイズ
```

修正の論理単位：

1. **YuNet letterbox 化**（`YuNetFaceDetector.swift`）
   - 640×640 stretch → アスペクト維持 letterbox + 黒パディング
   - bbox 逆変換に padX / padY / scale を反映
   - これで縦長/横長動画の歪みによる体の誤検出を排除

2. **eyeRatio 下限 0.35 → 0.40**（`MediaPipeFaceLandmarkerAdapter.swift` の `plausibilityEyeRatioRange`）
   - 補助検出器が体を ROI 化して MP IMG モードに流したとき、0.35〜0.40 で通過していた誤フィットを排除
   - 正当な横顔 (0.41) は維持

3. **補助検出器 bbox の前段ガード**（同上、`augmentWithBBoxDetector`）
   - 最小サイズ 0.04 × 0.04、アスペクト比 0.5〜1.5
   - ROI 再検出のコストごと無駄を削減

4. **プレビュー描画の時刻同期**（`MosaicPreviewController.swift` の `renderCurrentFrame`）
   - `copyPixelBuffer(forItemTime:itemTimeForDisplay:)` の第 2 引数で実フレーム時刻を取得
   - その時刻で `selectedLandmarks(at:)` を引く → 一拍遅れ解消

## CI（GitHub Actions）での検証

### セットアップ（初回のみ）

1. **Google Drive にサンプル動画 5 本をアップロード**
   - 任意のファイル名で OK（UUID 一致 + サイズ推測で workflow が `s1.mov`〜`s5.mov` にリネーム）
   - フォルダの共有を「リンクを知っている全員」+「閲覧者」に
2. **GitHub Secrets を登録**
   - Repo → Settings → Secrets and variables → Actions → New repository secret
   - Name: `SAMPLE_DRIVE_FOLDER_ID`
   - Value: Drive フォルダ URL `https://drive.google.com/drive/folders/[ID]` の `[ID]` 部分
3. workflow `.github/workflows/dvalid.yml` を main にマージ（または現在ブランチで実行）

### 起動方法

- GitHub → Actions → "Detection Accuracy (5本サンプル動画)" → Run workflow
- input で `backends` `videos` を絞れる（例：`videos=s3,s4` `backends=yunet` で 2 ジョブだけ）
- 全 15 ジョブ並列で実行、各ジョブのビルド+検証で ~3〜5 分／ジョブ → **全体 5〜10 分で完了**

### 結果の見方

- **Summary タブ**: 自動生成された結果表（hit/total/rate/baseline/lowCy）
- **Artifacts**: 各ジョブの `test.log` と `result.json` (JSON 1 行に集約)
- baseline 80% 未満に劣化したジョブは XCTAssert 失敗 → 赤マーク

### Drive ファイル名 → sN.mov のマッピング

workflow が以下の順で判定：
1. `copy_5B417FE2*.MOV` → `s4.mov`（UUID 確定）
2. `copy_324B1BD2*.MOV` → `s5.mov`（UUID 確定）
3. 残り 3 本をサイズで推測：
   - >50MB → `s1.mov`
   - >30MB → `s2.mov`
   - else  → `s3.mov`

ファイルが追加されたり名前が変わったら `.github/workflows/dvalid.yml` の `Map Drive filenames → sN.mov` ステップを更新。
